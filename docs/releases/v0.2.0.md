# v0.2.0

**Release Date:** 2026-07-16

Adds asset upload support, enabling audio-driven avatar video generation from locally produced narration files.

## Highlights

- New `asset` package for uploading audio, images, and video to HeyGen's asset service (`upload.heygen.com`, a separate host from the main API)

## Added

- `asset.Client` with `Upload(ctx, contentType, reader)` returning a hosted `Asset` whose URL is usable in other HeyGen APIs — for example as `video.VoiceInput{Type: "audio", AudioURL: ...}` for audio-driven video generation
- `asset.ContentType*` constants for the supported asset types: JPEG, PNG, MP4, WebM, MPEG audio (MP3)
- `asset.WithBaseURL` option for overriding the upload service URL (testing, proxies)
- `Client.Asset` facade field on the root SDK client
- `heygen.Client.RequestURL` for raw-body requests against absolute URLs, used by endpoints hosted outside the configured API base URL
- Unit tests covering asset upload (headers, body, error envelope) and retry body handling

## Fixed

- Retries now rewind the request body via `GetBody`, so a 429-retried POST resends the full payload instead of an empty body. Previously a rate-limited JSON POST (or asset upload) would silently retry with an empty body.
